Idea 乱码解决

今天被初学者问到 Idea 控制台乱码问题，就像这样

网上关于这个问题解决方法有好多种，有的好用，有的不好用，让这个问题的解决带上了玄学色彩，今天就来掰扯掰扯。

首先要明白乱码的本质是：

文件存储时使用的编码和读取文件使用的编码不一致

那么 idea 中在哪里出现了这种不一致呢？

1. 编译阶段

控制台编译

对于还没用 IDEA，在控制台手动编译 java 类的同学来说，若 java 源代码的文件编码为 UTF-8，这时如果我们手动编译这个类，没有主动指定编码的话

javac 会按操作系统默认字符编码读入这个文件，若两者不一致，则肯定乱码

操作系统默认字符编码每人可能不同，例如我的操作系统默认字符编码为 GBK，可以用命令 chcp 查看

解决方法是告诉 javac：【请你用 UTF-8 编码去读取这个 java 源文件，因为它的编码也是 UTF-8，你俩要一致】转换成命令就是

这回再去编译、运行 Test 类，乱码不见了。
